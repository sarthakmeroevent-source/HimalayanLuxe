# Strategic Approach & Design Mindset for Himalayan Luxe

## 1. Modern Development Methodology
- **Component-Driven Development:** Build isolated, reusable, and testable UI components. Follow the atomic design principles (atoms, molecules, organisms, templates, pages) where applicable.
- **Strict Typing:** Leverage TypeScript strictly across the codebase. Define clear interfaces and types for all props, state, and API responses to catch errors at compile time and improve overall predictability.
- **State Management:** Keep local state within components and use context or lightweight global state solutions (e.g., Zustand or Jotai) only when cross-component state sharing is absolutely necessary. Avoid unnecessary re-renders.
- **Mobile-First & Responsive:** Guarantee a seamless experience across all devices. Use modern CSS/Tailwind methodologies to ensure responsive layouts that scale elegantly from mobile phones to high-resolution desktop displays.
- **Performance Optimization:** Maintain optimal performance metrics by utilizing dynamic imports/lazy loading, minimizing bundle sizes, optimizing images/videos, and leveraging proper caching strategies.
- **Version Control & CI/CD:** Practice clean Git workflows (feature branches, semantic commit messages) and ensure automated testing and linting are part of the deployment pipeline.

## 2. Modern Code Structure (Scalable & Proper)
- **Directory Organization:**
  - `/src/components`: Reusable UI elements (buttons, inputs, cards). Organize into logical subdirectories (e.g., `ui/`, `layout/`, `forms/`).
  - `/src/hooks`: Custom React hooks for encapsulating complex state or side-effects.
  - `/src/utils`: Helper functions, constants, formatting tools, and API clients.
  - `/src/assets`: Static files (images, fonts, global CSS/styles).
  - `/src/pages` or `/src/views`: Top-level route components representing complete screens.
- **Separation of Concerns:** Keep business logic separated from presentation logic. Use custom hooks to extract logic out of UI components.
- **Clean Code Principles:** Follow DRY (Don't Repeat Yourself) and SOLID principles. Ensure components do one thing and do it well. Make code self-documenting through clear, descriptive variable and function names.

## 3. Design Rules & Mindset (Vibrant yet Classic)
- **Aesthetic Core:** The goal is a "modern website with a rich and vibrant representation of a classic aesthetic." It should feel premium, timeless, and effortlessly elegant. Wait, it needs to be *wowing* at first glance.
- **Typography & Colors:** Use curated, highly contrasted, and luxurious typography (e.g., Playfair Display paired with a clean sans-serif like Manrope or Inter). Employ harmonious color palettes—such as deep, rich backgrounds contrasted with "liquid gold" accents or vibrant emerald/sapphire tones to elevate the classic feel into modern vibrancy.
- **Micro-Animations & Interactions:** A dynamic interface is a requirement. Utilize `framer-motion` for fluid state transitions, text reveals (like vaporizing effects and typewriter animations), and interactive hover effects (like glassmorphism buttons or cursor-following elements).
- **Layout & Dimension:** Maximize screen real estate. Use generous negative space (padding/margins) to let elements breathe. Move away from generic boxy layouts; embrace organic positioning, overlap, and parallax scrolling (via Lenis) to give a sense of depth and immersion.
- **Premium Sensibility & Quality:** Avoid anything generic or "minimum viable". Do not use placeholders without stylized intention; always ensure imagery (like silk backgrounds and cinematic videos) complements the high-end narrative of the brand.
- **Storytelling:** Every section of the site should invoke a feeling and narrate the brand’s luxury legacy. Ensure the UX guides the user naturally through this story without friction.
